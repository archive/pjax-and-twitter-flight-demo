Twitter Flight

A web framework

(client side JavaScript)

that is:

  • Event-driven
  • Component-based
  • Lightweight
  • Test/spec friendly
  • Mixins based, not inheritance
  • AOP influenced

The basics

  1. Create a component
  2. Attach it to a DOM node
  3. Rock on!

Some rules

  1. The component should not understand which DOM-node it will be attached to
  2. Never communicate directly between components, always use pus/sub
  3. Never touch anything outside the component

Create a component (header.js)


return Flight.defineComponent(function Header () {
	// Add some code :)
});


Attach it to a DOM node (app.js)


Header.attachTo('.header');

Pub / Sub


Subscribe to events


this.on(document, 'title:changed', this._update);


Publish events


var payload = { title: 'some title' };
this.trigger('title:changed', payload);

DOM events


this.defaultAttrs({
	searchButton: '.search-action'
});

this.after('initialize', function () {
	this.on('click', {
		searchButton: this._search
	});
});

* Delegation by default

Reading DOM values


this.defaultAttrs({
	searchText: '.search-text'
});

this._search = function () {
	var value = this.select('searchText').val();
	alert('search on ' + value);
};

* this.select = component context
